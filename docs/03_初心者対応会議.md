# 改善会議議事録 — 「0からの初心者が学べるCodeQuest」へ

**日時**: 2026年7月8日 10:00〜11:30
**議題**: ユーザーフィードバック「既に知識がある人の腕試しには良いが、0から初心者が学ぶにはだめ」への対応
**参加者**: 綾瀬ミナト(ゲームデザイン) / 白鳥レイカ(教育工学) / 九条ハヤテ(エンジニア) / 桜庭ユズ(UX)

---

## 1. 問題の直視

**白鳥**: 耳の痛い指摘だけど、完全に正しい。今のCodeQuestは**テスト(評価)はあるが、インストラクション(教授)がない**。教育設計の言葉で言えば「事前知識ゼロの学習者には、評価の前に提示(presentation)と例示(example)が必要」なのに、うちはいきなり4択を突きつけている。`let x = 5;` を一度も見たことがない人には、最初のバグスライムが実質ボスよ。

**桜庭**: データでも裏が取れる。想定ペルソナ「挫折しかけの初学者」より手前、「これから始める人」が遊ぶと、最初の戦闘で不正解→被弾→解説を読んでも文脈がない→2敗目で離脱、が目に見える。**解説は「2回目に間違えないための道具」であって「初めて学ぶための道具」じゃない**のよね。

**綾瀬**: ゲームデザインの観点でも欠陥だ。ドラクエで最初にスライムと戦うとき、プレイヤーは「たたかうを選べば剣を振る」ことを**村の人から教わってから**旅立つ。うちの勇者は剣の振り方を知らずに草原に放り出されている。

**九条**: では「学んでから戦う」を最短の実装でどう入れるかだ。大工事はしない。教育の骨格だけを確実に入れる。

## 2. 決定方針 — 「まなぶ → ためす → たすけあり でたたかう」

**白鳥**: 教育設計の原則を3層で入れる。

1. **提示(まなぶ)**: 各エリアに**「まなびの石碑」**を置く。戦う前にその分野の基礎を数ページのレッスンで教える。「変数=なまえつきの はこ」のような比喩→コード例→ミニ確認、の順。一度読んだレッスンはメニューの**「まなぶ」**からいつでも読み返せる。
2. **足場かけ(ヒント)**: バトルの出題中に**「ヒント」ボタン**を用意。答えそのものではなく考え方を示す(例:「+ は かたほうが もじれつ だと…?」)。初心者は詰まらない、でも使うと**その一撃のダメージが半分**になるコストで、卒業への動機も残す。
3. **導入(チュートリアル)**: 初めての戦闘で、長老の声がバトルの流れ(正解=攻撃/ヒントの使い方/間違えても大丈夫)をガイドする。

**綾瀬**: 石碑はゲーム的にも良い。「村の長老の教え」として世界観に溶ける。石碑を読むと**キラッと光って「まなんだ!」**の演出を入れよう。地図の宝箱カウンターの隣に**レッスン既読数**も出す。学習の進捗もコレクション要素にする。

**桜庭**: UX要件を3つ。(1) 石碑は**スポーン地点の目の前**に置いて必ず視界に入れる。未読のあいだは光らせて誘導する。(2) エリア初入場時に「石碑で まなぼう!」のメッセージを出す。(3) ヒントボタンはタップしやすいサイズで出題ウィンドウの右上に常設。使用は**無制限**(ダメージ半減コストがあるから濫用は自然に収まる)。

**九条**: 実装スコープを確定する。
- データ: `lessons.ts`(3レッスン×6ページ、ふりがな多め・比喩→例→確認の構成)、全36問に `hint` フィールド追加
- マップ: 新タイル `L`(石碑)を3エリアのスポーン付近に配置
- メニュー: 「まなぶ」を追加(レッスン一覧→ページビューア、既読マーク)
- バトル: ヒントボタン(クリック/Hキー)、ヒント使用時ダメージ半減、初回戦闘チュートリアル
- セーブ: `readLessons` / `tutorialSeen` を追加(旧セーブは自動補完)

**白鳥**: 問題データ自体も1点だけ手を入れたい。難易度1の問題文に**前置きの一言**は要らない——石碑で教えるから。ただしヒントは全問必須。「ヒントを読めば難易度1は必ず解ける」水準に書く。これが初心者の安全網になる。

## 3. やらないこと(スコープ外)

- 実コード記述エディタ(v2以降。まず読める→選べるを固める)
- 音声読み上げ・ふりがなルビ機能(効果は認めるが今回は文面をやさしくすることで代替)
- レッスンの動画/アニメ化(石碑のテキストで十分検証できる)

## 4. 成功の定義

**桜庭**: 「JS未経験者が、石碑→チュートリアル→ヒントありでバグスライム2体を倒し、1つめのボスまで自力で到達できる」。これを新しい最低保証ラインとする。

**綾瀬**: 承認。「まなぶ→ためす→たすけありで たたかう」——これでCodeQuestは腕試し場から**冒険しながら学べる教室**になる。実装へ。
