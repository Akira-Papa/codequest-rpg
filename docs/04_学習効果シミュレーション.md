# 学習効果の定量分析レポート — CodeQuest v1.2

**実施日**: 2026年7月8日
**目的**: 「0からの初心者がこのゲームで本当に学べているか」を、(1) 実機テストプレイ、(2) 学習者モデルによるシミュレーション(各シナリオ500回試行)で定量評価する。

---

## 1. 実機テストプレイ(手動検証)

新規セーブから初心者フローを実際にプレイして確認した。

**プレイ内容**: 石碑レッスン読了 → 初回バトル(長老チュートリアル3ページ) → わざと不正解(反撃-6・解説強制読了) → ヒント使用(ダメージ半減 11→5) → 正解を重ねて3コンボ(x2, 22ダメージ) → 勝利。

**プレイ後にゲーム内へ蓄積されていた学習データ(localStorage実測値)**:

```json
{
  "readLessons": ["lesson-variable"],
  "tutorialSeen": true,
  "battleWins": 1, "bestCombo": 3,
  "stats": {
    "v1-2": {"asked": 1, "correct": 1, "wrongCount": 0},
    "v2-1": {"asked": 1, "correct": 1, "wrongCount": 0},
    "v2-2": {"asked": 1, "correct": 1, "wrongCount": 0},
    "v2-3": {"asked": 1, "correct": 0, "wrongCount": 1},
    "v2-4": {"asked": 1, "correct": 1, "wrongCount": 0}
  }
}
```

- メニュー「つよさ」に **へんすう せいかいりつ 80% (4/5)** と正しく可視化された
- 間違えた `v2-3`(constとpush)は `wrongCount: 1` となり、出題重みが1→3倍に上がって**再出題されやすくなる**(間隔反復が実データで結線されていることを確認)

## 2. シミュレーションの方法

**エンジン**: 実際のゲームロジックをそのままimportして使用(`selectQuestion`の重み付き出題、`recordAnswer`、`comboDamage`、`resolveVictory/Defeat`、敵ステータス、推奨レベルでのボス挑戦ポリシー)。乱数はシード固定で再現可能。実行は `npm run simulate`([scripts/simulate.ts](../scripts/simulate.ts))。

**学習者モデル**(仮定値はスクリプト冒頭に明記):
- 各問題に習得度 m(=正答確率)。完全初心者は 0.25(4択の偶然水準)、経験者は 0.80。上限0.95(ケアレスミス残存)
- **レッスン読了**: 該当分野の習得度が上がる(難易度1: +40%・2: +20%・3: +10%、いずれも残余分に対して)
- **解説を読む**: 不正解時 +45%(1.5秒の強制読了)、正解時 +20%
- **ヒント**: その出題の正答確率を+55%底上げ+習得度+10%。習得度0.55未満のとき使用(「自信がないとき押す」行動モデル)
- 忘却なし(単一セッション想定。限界として§5に記載)

**シナリオ**: S1 = 初心者×v1.0相当(石碑・ヒントなし)/ S2 = 初心者×v1.2 / S3 = 経験者(参照基準)。各500回。

## 3. 結果

| 指標 | S1 初心者×v1.0 | S2 初心者×v1.2 | S3 経験者 |
|---|---|---|---|
| クリア率(400戦以内) | 100% | 100% | 100% |
| クリアまでの戦闘数 | 30.0 | **26.3** | 25.5 |
| **敗北回数** | 4.34 | **0.84 (-81%)** | 0.03 |
| 序盤の正答率(最初の20問) | 46.6% | **67.6% (+21.0pt)** | 83.1% |
| 終盤の正答率(最後の20問) | 80.7% | **84.5%** | 90.9% |
| 学習ゲイン(序盤→終盤) | +34.1pt | +16.9pt | +7.8pt |
| **初見の正答率** | 25.0%(偶然水準) | **65.4%** | 81.3% |
| 3回目以降の正答率 | 80.9% | 83.1% | 91.1% |
| 直前に間違えた問題の次回正答率 | 70.2% | 79.6% | 90.5% |
| ヒント使用率(初見→3回目以降) | ― | **64.1% → 0.0%** | ― |
| 最終習得度(全36問平均) | 73.9% | **74.5%** | 88.2% |

## 4. 解釈 — 「学べているか」への回答

**✅ 学べている。しかも v1.2 の教育機能は「学習量を落とさず、挫折要因だけを取り除いている」。**

1. **学習自体は両方式で成立している**: S1でも正答率46.6%→80.7%(+34.1pt)と上がる。解説の強制読了と間隔反復(間違えた問題の次回正答率70〜80%)が効いている。ゲームのコアループは学習装置として機能している。
2. **v1.2の価値は「入口の体験」**: 初見正答率が25%(完全な当てずっぽう)→65.4%。つまり石碑レッスンによって「初めて見る問題でも3問に2問は自力で解ける」状態で戦闘に入れる。**序盤正答率46.6%はほぼ2問に1問被弾する体験**であり、これが67.6%になることで敗北が4.34回→0.84回(-81%)に減る。学びの中身ではなく「学び始めの痛み」を取り除いたと言える。
3. **足場は自然に外れる(scaffolding fade)**: ヒント使用率は初見64.1%→3回目以降0.0%。ダメージ半減コストの設計どおり、習得が進むと学習者は自発的にヒントを卒業する。補助輪が外れる様子が数値で確認できた。
4. **最終習得度はほぼ同じ(73.9% vs 74.5%)**: ヒントに頼っても最終的な学びは減らない。「ヒント=答えではなく考え方」+「解説は全問読む」設計のため、補助が学習の代替ではなく入口になっている。
5. **学習ゲインが S2 < S1 なのは天井効果**: S2は序盤から高いところで始まるため伸び幅が小さく見えるだけで、終盤到達点はS2の方が高い(84.5% > 80.7%)。
6. **経験者(S3)の腕試し体験も維持**: 敗北ほぼゼロ・25戦でクリア。初心者対応が上級者体験を壊していない。

**成功基準(docs/03 §4)との照合**: 「JS未経験者が石碑→チュートリアル→ヒントありでボス1まで自力到達」— S2の敗北0.84回/クリア率100%はこれを大きく上回って達成。

## 5. 限界と今後

- 学習イベントの効果量(+45%等)は認知科学の一般的知見に基づく**仮定値**であり、実測ではない。次のステップは実ユーザーのstats(ゲームが既に記録している)をエクスポートして本レポートの予測と突き合わせること
- 忘却(翌日以降のリテンション)は未モデル化。間隔反復の真価は複数セッションで出るため、将来「日をまたいだ再訪時に苦手問題を優先出題」する設計が有効
- 宝箱・逃走・レッスン再読は行動ポリシーに含めていない(保守的な見積もり)

## 6. 再現方法

```bash
npm run simulate   # 3シナリオ × 500回、シード固定で本レポートと同じ数値が出る
```
